Always use self-documenting names for files, scripts, and variables — clarity over brevity.

**Why:** Generic names like `script.py` or `data.json` force the reader to open the file to understand its purpose. Descriptive names make the workspace navigable at a glance.

**How to apply:** When naming any new file, ask "would someone unfamiliar with this project understand what this file does from the name alone?"
