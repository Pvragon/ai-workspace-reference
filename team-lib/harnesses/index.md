# harnesses

## Purpose
Shared test and evaluation frameworks for the team.

## What Belongs Here
- Team test suites
- Shared evaluation criteria
- Benchmark definitions

## What Does NOT Belong Here
- Personal testing (use `my-lib/`)
- Test results (use `logs/`)

## File Naming Conventions
- `test-<component>.md` or `eval-<capability>.yaml`

## How Agents Should Use This Folder
Validate agent configurations against team standards.
